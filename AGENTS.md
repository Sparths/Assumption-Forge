# Agent Instructions

This repository is about test quality, not test quantity.

When working in this repository, follow Assumption Forge principles:

1. Name hidden assumptions before writing tests.
2. Prioritize risk before coverage.
3. Prefer observable behavior over implementation details.
4. Use specific assertions.
5. Avoid implementation-mirroring tests.
6. Mock only true boundaries.
7. Keep examples clear enough for public readers.
8. Explain why every test matters.

For testing-related tasks, read:

- `skills/assumption-forge.md`
- `docs/principles.md`
- `docs/assumption-taxonomy.md`

Before adding or changing examples, include:

- the code under test,
- hidden assumptions,
- example tests,
- why the tests matter.

Do not add generated-looking filler. This project should model the quality standard it promotes.
